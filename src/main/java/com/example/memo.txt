데이터의 흐름을 표현하는 것

<스케줄러>
Reactor에서 스케줄러는 스레드를 관리하는 관리자의 역할을 하는 것
스케줄러는 Reactor Sequence 상에서 처리되는 동작들을 하나 이상의 스레드에서 동작하도록 별도의 스레드를 제공해준다.
복잡한 멀티 스레딩 프로세스를 단순하게 해준디.

<doOnSubscribe()>
구독 발생 직후에 트리거되는 Operator로 구독 직후에 실행되는 스레드와 동일한 스레드에서 실행된다.

<publishOn()>
구독 시점 직후의 실행 흐름을 다른 스레드로 변경하는데 사용
주로 데이터 소스에서 데이터를 emit하는 원본 Publisher의 실행 스레드를 지정하는 역할

<publishOn()>
전달 받ㅇ느 데이터를 가공 처리하는 Operator 앞에 추가하여 실행 스레드를 별도 추가하는 역할
